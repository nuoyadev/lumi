## 🎯 PROJECT MISSION

**ghostpilot** = an AI-powered social media autopilot that:
1. Reads content ideas from a queue (`content/queue.json`)
2. Optionally scans GitHub commits for auto-content generation
3. Uses an LLM (Claude API or OpenAI) to generate platform-native posts
4. Uses **Playwright** (headless browser) to post on **Twitter/X** and **Instagram**
5. Runs automatically every morning via **GitHub Actions** (free, zero infrastructure)
6. Costs **$0** for posting — no official Twitter/Instagram API needed

**Core principle:** Build silently. Post loudly.

        ## 📋 CONTENT QUEUE FORMAT

        ### `content/queue.json`
        ```json
        [
          {
              "id": "post_001",
                  "topic": "Just shipped MCP-native routing in cerocrates — agents that never hallucinate on tools",
                      "platforms": ["twitter", "instagram"],
                          "tone": "technical",
                              "used": false,
                                  "created_at": "2026-05-23"
                                    },
                                      {
                                          "id": "post_002",
                                              "topic": "Why 40% of AI agents fail silently — and the 3-line fix",
                                                  "platforms": ["twitter"],
                                                      "tone": "opinion",
                                                          "used": false,
                                                              "created_at": "2026-05-23"
                                                                }
                                                                ]
                                                                ```

                                                                ### Queue rules:
                                                                - Posts with `"used": true` are skipped
                                                                - If all posts are used, ghostpilot logs a warning and exits gracefully
                                                                - After posting, set `"used": true` and add `"posted_at": "ISO timestamp"`
                                                                - Never delete entries — keep history

                                                                ## 🧩 MODULE SPECIFICATIONS

                                                                ### 1. `src/generator.py`

                                                                **Purpose:** Take a topic/tone and return platform-native content.

                                                                **Interface:**
                                                                ```python
                                                                async def generate_post(
                                                                    topic: str,
                                                                        platform: str,          # "twitter" | "instagram"
                                                                            tone: str,              # "builder" | "technical" | "opinion" | "story"
                                                                                provider: str = "claude"
                                                                                ) -> str:
                                                                                    """
                                                                                        Returns generated post content as a string.
                                                                                            Twitter: max 280 chars, punchy, no hashtag spam
                                                                                                Instagram: storytelling, 3-5 paragraphs, 5-10 relevant hashtags at end
                                                                                                    """
                                                                                                    ```
                                                                                                    
                                                                                                    **LLM Prompts — use exactly these system prompts:**
                                                                                                    
                                                                                                    For Twitter:
                                                                                                    ```
                                                                                                    You are a ghostwriter for a developer who builds AI systems and ships in public.
                                                                                                    Write a tweet that is sharp, direct, and sounds human — not like AI.
                                                                                                    Max 260 characters. No emojis spam. No "excited to announce". No corporate speak.
                                                                                                    One clear idea. Punchy ending.
                                                                                                    ```
                                                                                                    
                                                                                                    For Instagram:
                                                                                                    ```
                                                                                                    You are a ghostwriter for an AI developer who ships in public.
                                                                                                    Write an Instagram caption that tells a short story or shares a real insight.
                                                                                                    3-4 short paragraphs. Human tone. End with 5-8 relevant hashtags on a new line.
                                                                                                    No fake motivation. No "hustle culture" clichés.
                                                                                                    ```
                                                                                                    
                                                                                                    **Error handling:**
                                                                                                    - If API call fails: retry once after 3 seconds, then raise `GeneratorError`
                                                                                                    - If response is empty: raise `GeneratorError("LLM returned empty response")`
                                                                                                    - Log all API calls with token count if available

                                                                                                    ### 2. `src/poster_twitter.py`
                                                                                                    
                                                                                                    **Purpose:** Open Twitter/X in a headless Playwright browser and post a tweet.
                                                                                                    
                                                                                                    **Interface:**
                                                                                                    ```python
                                                                                                    async def post_tweet(content: str, dry_run: bool = False) -> bool:
                                                                                                        """
                                                                                                            Posts a tweet. Returns True on success, False on failure.
                                                                                                                If dry_run=True, prints the content but does NOT post.
                                                                                                                    """
                                                                                                                    ```
                                                                                                                    
                                                                                                                    **Playwright flow (implement exactly this):**
                                                                                                                    ```
                                                                                                                    1. Launch chromium (headless=True in CI, headless=False for local debug)
                                                                                                                    2. Navigate to https://x.com/login
                                                                                                                    3. Wait for username field → type username (slow, human-like)
                                                                                                                    4. Click Next button
                                                                                                                    5. Wait for password field → type password (slow, human-like)
                                                                                                                    6. Click Login button
                                                                                                                    7. Wait for home feed to load (wait for tweet compose area)
                                                                                                                    8. Click tweet compose box
                                                                                                                    9. Type content character by character with random delay (10-30ms)
                                                                                                                    10. Wait 1-2 seconds (human pause)
                                                                                                                    11. Click "Post" button
                                                                                                                    12. Wait for success confirmation (tweet appears in feed)
                                                                                                                    13. Take screenshot → save to logs/screenshots/twitter_{timestamp}.png
                                                                                                                    14. Close browser
                                                                                                                    ```
                                                                                                                    
                                                                                                                    **Critical rules:**
                                                                                                                    - Use `page.wait_for_selector()` with reasonable timeouts (15s max)
                                                                                                                    - Add `asyncio.sleep(random.uniform(0.5, 1.5))` between each major action
                                                                                                                    - Never use `page.fill()` directly — simulate human typing with `page.type()`
                                                                                                                    - On any `TimeoutError`: take a screenshot, log the error, return False
                                                                                                                    - Store browser state (cookies/session) in `logs/sessions/twitter_session.json` to avoid re-login

                                                                                                                                        **Purpose:** Scan recent GitHub commits and turn them into content ideas.
                                                                                                                                        
                                                                                                                                        **Interface:**
                                                                                                                                        ```python
                                                                                                                                        async def scan_recent_commits(
                                                                                                                                            username: str,
                                                                                                                                                repo: str,
                                                                                                                                                    limit: int = 5
                                                                                                                                                    ) -> list[dict]:
                                                                                                                                                        """
                                                                                                                                                            Returns a list of content ideas generated from recent commits.
                                                                                                                                                                Uses GitHub's public API (no auth needed for public repos).

                                                                                                                                                                        **Logic:**
                                                                                                                                                                        - Call `https://api.github.com/repos/{username}/{repo}/commits?per_page={limit}`

                                                                                                                                                                        - For each meaningful commit, ask the LLM to generate a content idea
                                                                                                                                                                        - Return a list ready to be injected into the queue
                                                                                                                                                                        - Only use commits from the last 7 days

                                                                                                                                                                        **Purpose:** Main entry point. Orchestrates everything.
                                                                                                                                                                        
                                                                                                                                                                        **CLI interface:**
                                                                                                                                                                        ```bash
                                                                                                                                                                        python src/scheduler.py                    # normal run: pick next queued post, post it
                                                                                                                                                                        python src/scheduler.py --dry-run          # generate content, print it, don't post
                                                                                                                                                                        python src/scheduler.py --post-now "topic" # generate + post immediately from a topic
                                                                                                                                                                        python src/scheduler.py --scan-github      # scan commits and add to queue

                                                                                                                                                                        **Main flow:**
                                                                                                                                                                        ```python
                                                                                                                                                                        async def main():
                                                                                                                                                                            1. Load config
                                                                                                                                                                                2. Initialize logger
                                                                                                                                                                                    3. Parse CLI args
                                                                                                                                                                                        4. If --scan-github: run github_scanner, append to queue, exit
                                                                                                                                                                                            5. If --status: print queue summary, exit
                                                                                                                                                                                                6. Load queue.json
                                                                                                                                                                                                    7. Find next unused post
                                                                                                                                                                                                        8. If none: log warning "Queue is empty. Add posts to content/queue.json", exit
                                                                                                                                                                                                            9. Generate content for each platform in post["platforms"]
                                                                                                                                                                                                                10. If not dry_run: post on each platform
                                                                                                                                                                                                                    11. Mark post as used + add posted_at timestamp
                                                                                                                                                                                                                        12. Save queue.json
                                                                                                                                                                                                                            13. Log summary
                                                                                                                                                                                                                            ```
